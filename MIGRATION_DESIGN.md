## 1. 기존 UI 구조 분석 (Streamlit)

### 1-1. 화면 구성

| 영역 | 구현 | 내용 |
|------|------|------|
| **Sidebar (좌측)** | `st.sidebar` | 입력 폼: 년/월/일, 달력 유형, 출생 시간, 성별, 이름, 분석 버튼 |
| **Main (중앙)** | `st.markdown` | 타이틀("🔮 사주명리 분석"), 서브타이틀 |
| **Tabs (4개)** | `st.tabs` | 📊 분석 요약 / 📋 상세 분석 / 📅 대운 타임라인 / ℹ️ 사주 정보 |
| **Sub-Tabs** | `st.tabs` 중첩 | 탭2 내부: 십신/12운성/합충/신강신약 — 탭4 내부: 사주란?/천간지지/오행/십신/12운성 |

| **Footer** | `st.caption` + 커스텀 div | 면책 문구(노란 배경 박스), 버전 정보 |

## 2. 화면 재설계 (Next.js)

### 2-1. 페이지 구조

| 페이지 | URL | 역할 |
|--------|-----|------|
| **홈** | `/` | 서비스 소개 + 입력 폼 (사이드바 없이 중앙 배치) + 사주 정보 (미분석 상태) |
| **분석 결과** | `/analysis` | 분석 결과 4개 탭 (URL 파라미터 없이, 상태 기반) |
| **사주 정보** | `/info` | 사주란?, 천간·지지, 오행, 십신, 12운성 (독립 페이지) |

### 2-2. 라우팅 구조

```
/                          → 홈 (입력 폼 + 소개)
/analysis                  → 분석 결과 (분석 데이터 없으면 / 로 리다이렉트)
/analysis?tab=summary      → 분석 요약 탭 (쿼리 파라미터로 탭 상태 유지)
/analysis?tab=detail       → 상세 분석 탭
/analysis?tab=luck         → 대운 타임라인 탭
/analysis?tab=info         → 사주 정보 탭
/info                      → 사주 정보 (교육 콘텐츠)
/info/stems-branches       → 천간·지지 상세
/info/elements             → 오행 상세
/info/ten-gods             → 십신 상세
/info/twelve-stages        → 12운성 상세
```

## 3. 레이아웃 설계

### 3-1. 전체 레이아웃

```
┌──────────────────────────────────────────────┐
│  Header (고정)                                │
│  [🔮 사주명리 분석] [정보] [GitHub]            │
├───────────┬──────────────────────────────────┤
│ Sidebar   │  Content                         │
│ (280px)   │                                  │
│           │  [Tab1] [Tab2] [Tab3] [Tab4]     │
│ 입력 폼    │                                  │
│ - 년월일   │  {{ 탭 내용 }}                    │
│ - 달력     │                                  │
│ - 시간     │                                  │
│ - 성별     │                                  │
│ - 이름     │                                  │
│ [분석하기]  │                                  │
│           │                                  │
│ 면책 안내  │                                  │
├───────────┴──────────────────────────────────┤
│  Footer (사이트 정보, 버전, 면책 요약)          │
└──────────────────────────────────────────────┘
```

### 3-2. 반응형 기준

| Breakpoint | 범위 | 레이아웃 변화 |
|-----------|------|-------------|
| **Mobile** | < 768px | Sidebar → 하단 Drawer (버튼으로 열기), 카드 1열, 대운 카드 2열, 원국표 가로 스크롤 |
| **Tablet** | 768px~1024px | Sidebar → 축소 가능(아이콘만), 카드 2열, 대운 카드 3열 |
| **Desktop** | > 1024px | Sidebar 280px 고정, 카드 4열, 대운 카드 5열 |

## 5. 인터랙션 설계

### 5-1. 사용자 입력 흐름

1. 사이드바에서 년/월/일을 `number input`으로 입력
2. 달력 유형 `RadioGroup`에서 "양력"/"음력" 선택 → "음력" 선택 시 윤달 체크박스 노출
3. "시간을 아십니까?" `RadioGroup` → "예" 선택 시 슬라이더(0~23) 노출
4. 성별 `RadioGroup` → "선택 안함" 시 대운 미산출 안내 문구 표시
5. 이름 텍스트 입력 (선택)
6. "🔍 사주 분석하기" 버튼 클릭

### 5-2. 버튼 클릭 동작

| 동작 | 상세 |
|------|------|
| **클라이언트 유효성 검사** | `zod` 스키마로 즉시 검사. 실패 시 해당 필드 아래에 빨간 텍스트 에러 표시 |
| **API 호출** | `useMutation`으로 `POST /api/analysis` 호출 |
| **로딩 중** | 버튼 텍스트 → "분석 중..." + Spinner, 폼 전체 `disabled` |
| **성공** | Toast("✅ 분석 완료!"), `/analysis` 페이지로 `router.push`, React-Query 캐시에 결과 저장 |
| **실패 (400)** | Toast("❌ " + 에러 메시지), 폼 재활성화 |
| **실패 (500)** | Toast("서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요."), 폼 재활성화 |

### 5-3. 로딩 처리 방식

| 상황 | UI |
|------|-----|
| 분석 API 호출 중 | 사이드바: 버튼 Spinner + disabled / 메인: 전체 Skeleton UI (원국표 모양 + 카드 4개 + 차트 영역) |
| 참조 데이터 로딩 중 | 정보 탭: 테이블 Skeleton (행 5줄 회색 바) |
| 탭 전환 | 즉시 전환 (데이터 이미 메모리에 있음), 별도 로딩 없음 |

### 5-4. 에러 표시 방식

| 에러 유형 | 표시 방식 |
|----------|----------|
| 폼 유효성 실패 | 해당 입력 필드 하단에 빨간 텍스트 (`<p className="text-red-500 text-sm">`) |
| API 400 에러 | `sonner` Toast (error variant, 상단 우측, 5초 유지) |
| API 500 에러 | Toast + 메인 콘텐츠에 `<Alert variant="error">` 표시 |
| 네트워크 에러 | Toast("네트워크 연결을 확인해주세요.") |

## 7. UX 개선 포인트

### 7-1. Streamlit 대비 개선 항목

| # | 개선 항목 | Streamlit 한계 | Next.js 개선 |
|---|----------|---------------|-------------|
| 1 | **페이지 리로드 없음** | 버튼 클릭 시 전체 스크립트 재실행 → 화면 깜빡임 | SPA, 상태 유지, 탭 전환 즉시 |
| 2 | **URL 공유** | 분석 결과를 URL로 공유 불가 | 쿼리 파라미터로 탭 상태 유지, 향후 결과 ID 기반 공유 가능 |
| 3 | **모바일 대응** | Streamlit 기본 반응형은 사이드바가 모바일에서 불편 | Drawer 패턴, 터치 최적화, 카드·테이블 반응형 |
| 4 | **로딩 UX** | `st.spinner`는 단순 텍스트 | Skeleton UI로 결과 형태를 미리 보여줌 → 체감 로딩 시간 단축 |
| 5 | **에러 안내** | `st.error`로 인라인 표시, 사라지지 않음 | Toast 자동 소멸 + 필드별 인라인 에러 분리 |
| 6 | **차트 인터랙션** | `st.bar_chart`는 hover 정보 제한적 | recharts 커스텀 Tooltip (오행 이름, 수치, 퍼센트, 색상) |
| 7 | **원국표 시각성** | HTML 문자열 기반, 스타일 제한 | React 컴포넌트로 hover 시 천간/지지 정보 팝오버, 오행 색상 배경 적용 |
| 8 | **대운 카드** | HTML 문자열 기반 정적 카드 | hover 시 상세 설명 팝오버, 현재 대운 스크롤 자동 포커스 |
| 9 | **접근성** | 사실상 미지원 | `aria-label`, 키보드 내비게이션, 포커스 관리, 스크린리더 대응 |
| 10 | **SEO** | Streamlit은 SPA이므로 SEO 불가 | 정보 페이지는 SSG/SSR로 검색 엔진 노출 가능 |

### 7-2. 사용성 개선 방향

- **입력 기억**: `localStorage`에 마지막 입력값 저장 → 재방문 시 자동 복원
- **키보드 접근**: Tab 키로 입력 필드 순회, Enter 키로 분석 실행
- **인쇄 최적화**: `@media print` CSS로 결과 페이지 인쇄 시 사이드바 숨김 + 깔끔한 레이아웃
- **다크모드**: Tailwind `dark:` 유틸리티 클래스로 다크모드 지원 (갈색 → 따뜻한 다크 브라운)
- **애니메이션**: 탭 전환 시 `framer-motion` fade-in, 대운 카드 stagger 애니메이션, 차트 진입 애니메이션
